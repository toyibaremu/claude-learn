What are skills?

Skills are custom slash commands you create. They're Markdown files that contain instructions — when you type the skill name with a /, Claude Code reads those instructions and uses them as context for the current task.

💡
Think of skills as macros for your AI assistant Just like you might create a Stata macro or R function for something you do repeatedly, a skill packages up a set of instructions into a single command you can reuse forever.

Where skills live

my-project/
├── CLAUDE.md
└── .claude/ ← Claude Code config folder
    └── commands/ ← Your custom skills go here
        ├── balance-table.md
        ├── summarise-data.md
        └── referee-response.md

Once a skill file exists, you use it by typing /balance-table (or whatever you named it) in your session.

Creating your own skills

A skill file is just a Markdown file with instructions. Here's how to create one:

# Create the directory
mkdir -p .claude/commands

# Create your skill file
# (Claude Code will help you write the content)
claude
> Create a skill called "balance-table" that generates a
  standard balance table comparing treatment and control groups.
  Save it to .claude/commands/balance-table.md

Or write it manually. Here's what a skill file looks like:

# .claude/commands/balance-table.md

Create a balance table for the current project's main dataset.

Follow these steps:
1. Look at the CLAUDE.md to identify the main dataset and key variables
2. Load the dataset
3. Create a table comparing treatment vs. control groups for all key variables
4. Include: mean, SD, and p-value of t-test for each variable
5. Format with stars: * p<0.10, ** p<0.05, *** p<0.01
6. Export to output/tables/balance_table.tex and balance_table.xlsx
7. Report how many observations are in each group

Use the statistical software defined in CLAUDE.md.
Follow the coding conventions in CLAUDE.md.

Research skills examples

Inspired by Scott Cunningham's MixtapeTools and Pedro Sant'Anna's workflow, here are skills that researchers commonly create:

/data-audit — Audit a dataset for issues

Perform a comprehensive data audit on the main dataset.
Check for:
- Missing values (by variable, report % missing)
- Duplicates (by key identifier)
- Outliers (values beyond 3 SD from mean for continuous variables)
- Implausible values (negative ages, incomes above $1M, etc.)
- Variables with near-zero variance (might be errors)
Output a formatted report to logs/data_audit.txt

/summarise-data — Quick data summary

Load the main dataset and produce:
1. A table with N, mean, SD, min, max for all numeric variables
2. Frequency tables for all categorical/string variables
3. A note about any obvious data quality issues
Print to screen and save to output/tables/data_summary.tex

/draft-results — Draft results section

Read the main regression results table (output/tables/main_results.tex)
and the balance table (output/tables/balance_table.tex).
Draft a "Results" section for the paper (~600 words).
Rules:
- Only state what the numbers actually show
- Don't make causal claims beyond what's identified
- Reference tables with \ref{} using the LaTeX labels
- End with a robustness check subsection header

/improve-writing — Polish academic prose

Read the file specified after this command.
Improve the writing by:
1. Cutting sentences longer than 30 words in half
2. Replacing passive voice where possible
3. Removing hedge words (arguably, seemingly, quite)
4. Making the first sentence of each paragraph the main point
Show tracked changes (before → after) for every edit.

/check-parallel-trends

Check the parallel trends assumption for my DiD specification.
1. Load the panel dataset from CLAUDE.md
2. Create an event study plot showing pre-treatment trends
3. Run a formal test for pre-trends using the 2 pre-periods
4. Report: are pre-trends parallel? What does the test say?
5. Save the event study plot to output/figures/event_study.pdf
Use the estimator specified in CLAUDE.md (default: fixest::feols)

Sharing skills with your team

Because skills are just files in your .claude/commands/ folder, they're automatically shared when you share your project via Git. Your whole lab can use the same skills with zero setup.

🌟
Global skills (available in all projects) Put skills in ~/.claude/commands/ (your home folder) to make them available in every project you work on. Great for things like /improve-writing or /data-audit that you want everywhere.
🎯

Module 6 Exercise: Build your first skill

Create two custom skills for your research workflow:

  1. Create .claude/commands/ in one of your project folders.
  2. Ask Claude to help you write a skill called summarise-data that generates a summary statistics table for your main dataset.
  3. Create a second skill for something you do regularly (e.g., draft-abstract, check-regression, or clean-column-names).
  4. Test both skills by typing /summarise-data in a Claude Code session.
