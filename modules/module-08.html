🔌 MCP — Connecting Claude to the World

The Model Context Protocol (MCP) lets you connect Claude Code to external tools, databases, and services — giving it superpowers beyond your local files.

What is MCP?

📖
Plain English: what is MCP? By default, Claude Code can only work with files on your computer. MCP is a standard way to connect Claude to other things: a database, a website, GitHub, Google Drive, Zotero, your calendar — anything you can write a connector for. Think of MCP servers as plugins that give Claude new abilities.

How MCP works

An MCP server is a small program that runs on your computer (or a remote server) and exposes tools that Claude can call. For example:

  • A database MCP server exposes tools like query_database(sql) and list_tables()
  • A GitHub MCP server exposes tools like create_pr(title, body) and list_issues()
  • A Zotero MCP server exposes tools like search_library(query) and get_citation(key)

You configure which servers to run in ~/.claude/settings.json, and Claude automatically has access to their tools.

Adding MCP servers

You add MCP servers to your settings file:

// ~/.claude/settings.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/you/research"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "your_token_here"
      }
    }
  }
}

Or add them via the command line (easier for beginners):

# Add the official Anthropic filesystem server
claude mcp add filesystem npx @modelcontextprotocol/server-filesystem ~/research

# Add a GitHub server
claude mcp add github npx @modelcontextprotocol/server-github

# List all configured servers
claude mcp list

Building a simple MCP server

You can build your own MCP server to connect Claude to any tool in your research stack. Here's a minimal example in Python:

# A simple MCP server that reads Stata datasets
from mcp.server import Server
from mcp.server.models import InitializationOptions
import mcp.types as types

app = Server("stata-reader")

@app.list_tools()
async def handle_list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="read_stata",
            description="Read a Stata .dta file and return summary info",
            inputSchema={
                "type": "object",
                "properties": {
                    "filepath": {"type": "string"}
                }
            }
        )
    ]

@app.call_tool()
async def handle_call_tool(name: str, arguments: dict):
    if name == "read_stata":
        import pandas as pd
        df = pd.read_stata(arguments["filepath"])
        return [types.TextContent(
            type="text",
            text=str(df.describe())
        )]

Module 8 Exercise: Add your first MCP server

Add the official filesystem MCP server and use it:

  1. Run: claude mcp add filesystem npx @modelcontextprotocol/server-filesystem ~/Documents
  2. Start a Claude Code session and ask: "Using the filesystem server, list all .csv files in my Documents folder"
  3. Try: "Find all .do files (or .R or .py files) in my Documents and summarise what research projects I have"
  4. If you have a SQLite database, add that server too and try a natural language query
