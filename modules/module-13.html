Habits of mastery

The researchers who get the most out of Claude Code share these habits:

✅ Masters do this
  • Always verify Claude's outputs before using them
  • Maintain good CLAUDE.md files for every project
  • Ask Claude to critique its own work
  • Use Git so every change is reversible
  • Build a personal library of custom skills
  • Think in terms of workflows, not one-off questions
  • Update their approach as Claude Code improves
❌ Beginners do this
  • Accept outputs without checking
  • Re-explain the project every session
  • Ask vague, one-line questions
  • Work without version control
  • Repeat the same prompts manually each time
  • Treat Claude as a search engine, not a collaborator
  • Stick to what they learned in month one

The mastery mindset

"The question is not 'can I trust Claude?' The question is 'have I set up Claude to succeed?' A well-briefed Claude with a good CLAUDE.md, in a well-organised project, with a researcher who reviews outputs critically — that's a powerful combination."

Academic integrity and disclosure

⚠️
Your responsibility as a researcher You are responsible for every claim, result, and word in your published work — regardless of who or what helped you produce it. AI assistance does not reduce your responsibility for accuracy.

The four questions every researcher should ask

QuestionGuidance
Did Claude help write any part of this paper? Disclose in acknowledgements or per journal policy. Most journals now require disclosure of AI writing assistance.
Did Claude run or write any analysis code? Review all code carefully. You are responsible for its correctness. Share the code so others can verify it.
Are all citations accurate? Never trust Claude's citations without verifying them. Claude can hallucinate paper details. Always check against the actual source.
Could this qualify as research misconduct? Check your institution's AI policy and relevant journal policies. Policies are evolving rapidly — what's acceptable today may change.

Sample disclosure language

When disclosure is appropriate, here is example language adapted from various journal policies:

"The authors used Claude Code (Anthropic) to assist with [data cleaning / coding analysis scripts / drafting portions of this manuscript]. All outputs were reviewed and verified by the authors, who take full responsibility for the accuracy and integrity of this work."

Data privacy and security

🔒
Sensitive data: what NOT to send to Claude Code
  • PII (personally identifiable information): real names, national IDs, phone numbers, exact addresses
  • Health data: medical records, HIV status, mental health information
  • Financial records: individual bank records, tax filings
  • Protected data under research ethics approval: data you agreed to keep confidential

Safe alternatives: Anonymise data first, work with aggregated statistics, or use synthetic data with the same structure.

To work safely with sensitive data:

# GOOD: Share only the structure, not the content
> Here is the structure of my dataset (not the actual data):
  hh_id (int), village_id (int), treatment (0/1),
  income (float), age (int), education_years (int)
  Write a data cleaning script assuming this structure.

# GOOD: Anonymise before sharing
> I've replaced all real names with codes (HH001, HH002...).
  Now help me analyse this anonymised dataset.

AI access and global equity

Claude Code is potentially transformative for researchers in low- and middle-income countries who may not have access to large research labs, expensive software, or many RAs. But access isn't equal — here are some considerations:

Options: (1) Use the free tier of Claude.ai for lighter tasks. (2) Apply for Anthropic's research access programs. (3) Use the cheapest models (Haiku) for batch tasks. (4) Share costs with colleagues by pooling an API key. (5) Some universities now provide institutional Claude access.

Claude Code works on any internet connection — it sends text, not large files. Even a slow mobile data connection (1–2 Mbps) is sufficient. Once Claude has read your files, subsequent interactions are fast. Use /compact to keep conversations short and reduce data usage.

Claude was trained primarily on English-language, Western-context text. Be aware that: (1) Its knowledge of local context (specific country institutions, local languages, informal economies) may be limited. (2) It may default to Western methodological conventions. (3) For local-language data, accuracy may be lower. Always apply your domain expertise to supplement Claude's knowledge.
